Proceso de Ejecución de una Instrucción desde Memoria

Explicación del proceso:

1. Inicio: Comienza el ciclo de ejecución de instrucciones.

2. Cargar contador de programa (PC): Se inicializa el registro PC con la dirección de memoria de la primera instrucción a ejecutar.

3. Fetch: La CPU utiliza el bus de sistema para acceder a la memoria y lee la instrucción almacenada en la dirección indicada por el PC.

4. Incrementar PC: Se aumenta el valor del PC para que apunte a la siguiente instrucción en secuencia.

5. Decodificar instrucción: La unidad de control analiza el código de operación para determinar qué operación realizar y qué recursos necesitará.

6. Obtener operandos: Si la instrucción requiere datos adicionales, se accede a registros o memoria para obtenerlos.

7. Ejecutar instrucción: La ALU u otra unidad funcional ejecuta la operación especificada.

8. Repetir ciclo: El proceso se repite para la siguiente instrucción, continuando el flujo de ejecución.

Interconexión y Jerarquía de Buses

Jerarquía de buses:

Analizando la imagen mostrada en la diapositiva, se identifica una estructura jerárquica de buses diseñada para conectar dispositivos con diferentes velocidades de operación.

1. Bus del Sistema (Bus Sistema):
  • Es el bus principal que conecta componentes críticos y rápidos como el Procesador, Caché, Memoria y el Coprocesador Matemático.
  • Opera a la mayor velocidad del sistema.
  • Representa la columna vertebral de comunicaciones del computador.
2. Bus Local (Bus Local):
  • Conecta al procesador con dispositivos cercanos que requieren un acceso rápido, como la caché.
  • Utiliza una interfaz especializada para la comunicación.
  • Reduce la carga en el bus del sistema para operaciones específicas.
3. Bus de Alta Velocidad:
  • Permite la conexión de dispositivos de E/S de rendimiento medio-alto.
  • Utiliza interfaces con bus de alta velocidad y controladores especializados.
  • Sirve como puente entre el bus del sistema y los periféricos más rápidos.
4. Bus de Expansión:
  • Conecta dispositivos periféricos más lentos como el Controlador de Disco, Adaptadores de Vídeo, Red, Monitores y ranuras de expansión.
  • Posee interfaces de expansión para diferentes tipos de dispositivos.
  • Como se observa en la imagen, permite la conexión de múltiples periféricos a través de interfaces específicas.
5. Bus de E/S Externo:
  • El nivel más bajo de la jerarquía, dedicado a interconectar dispositivos externos.
  • Conecta periféricos como discos externos, impresoras, módems y ratones.
  • Incluye adaptadores específicos como Fax/módem, Puerto serie y Puerto paralelo para la conexión de dispositivos lentos.
  • Permite la interconexión de discos externos y otros dispositivos periféricos a través de interfaces especializadas.
Interfaces y Adaptadores:
  • Para conexión de dispositivos de E/S de alta velocidad: La imagen muestra interfaces especializadas que permiten la comunicación entre el bus del sistema y los dispositivos de alta velocidad.
  • Para conexión de dispositivos de E/S lentos: Se observan adaptadores específicos en el bus de expansión que facilitan la conexión de dispositivos de menor velocidad.
  • Interfaz con bus de E/S externo: Permite la comunicación con dispositivos periféricos externos al sistema.
Conclusión:

La jerarquía de buses presentada en la imagen permite una gestión eficiente de la comunicación entre componentes que operan a diferentes velocidades. Esta estructura estratificada optimiza el rendimiento del sistema al evitar que los dispositivos más lentos reduzcan la velocidad de los componentes más rápidos, asegurando que cada tipo de dispositivo se conecte al bus que mejor se adapta a sus requisitos de rendimiento.
