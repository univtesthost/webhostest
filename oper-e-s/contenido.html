Técnicas posibles para las operaciones de Entrada/Salida (E/S)

Cuadro Comparativo

Técnica Descripción Ventajas Desventajas
E/S Programada El CPU ejecuta instrucciones para verificar y realizar operaciones de E/S directamente. - Fácil de implementar
- Control total por el CPU 		- Poco eficiente
- El CPU espera sin poder hacer otras tareas
E/S por Interrupciones Los dispositivos generan interrupciones para notificar al CPU cuando están listos. - Mejora la eficiencia del CPU
- Respuesta rápida a eventos externos 		- Mayor complejidad en el manejo
- Requiere hardware para interrupciones
DMA (Acceso Directo a Memoria) Permite que los datos se transfieran directamente entre memoria y dispositivos sin intervención del CPU. - Alta eficiencia en la transferencia de datos
- Libera al CPU durante las transferencias 		- Requiere controlador DMA
- Más costoso y complejo

Resumen Sistémico Visual

E/S Programada

CPU: Controla todo el proceso.

Rendimiento: Bajo, por espera activa.

Ideal para: Sistemas simples.

E/S por Interrupciones

CPU: Atiende cuando se le avisa.

Rendimiento: Medio, permite multitarea.

Ideal para: Sistemas interactivos.

DMA

CPU: No participa directamente.

Rendimiento: Alto, por transferencia directa.

Ideal para: Transferencias masivas de datos.
